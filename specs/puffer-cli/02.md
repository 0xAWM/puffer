# Daemon yolo mode

## Purpose

Daemon deployments can launch `puffer daemon --yolo` when the outer runtime has
already decided the session should run without approval prompts or
workspace-only filesystem enforcement.

## Behavior

- `--yolo` has the compatibility alias
  `--dangerously-bypass-approvals-and-sandbox`.
- The flag is daemon-local process state and is not persisted to session
  metadata or workspace config.
- Each daemon agent turn applies yolo after loading the session record and
  before model or tool execution begins.
- Yolo sets the turn `AppState` sandbox mode to `danger-full-access` and grants
  session-wide allow-all permissions.

## Compatibility

Normal daemon behavior is unchanged when the flag is omitted. Existing
permission prompts and workspace-write path restrictions remain the default.
